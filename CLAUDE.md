# Warranty — Модуль фото и видео (мобильное приложение механика)

> Контекст проекта для Claude. Читается автоматически в начале каждой сессии.
> Версия ТЗ: 1.0 (черновик, май 2026).

## 1. Что это

Android-приложение для механика, которое документирует гарантийный случай **без ручного ввода данных**:

1. Механик нажимает «Начать осмотр».
2. Фотографирует номерной знак — первое и обязательное действие.
3. OCR распознаёт номер и создаёт папку кейса с названием = номер авто.
4. Все последующие фото/видео автоматически попадают в эту папку.
5. Механик отправляет «закончил» → папка закрывается, изменения невозможны.

Ключевой принцип: минимум действий механика, всё происходит автоматически.

## 2. Стек и платформа

- **Платформа:** Android, минимум **Android 10** (API 29).
- **Фреймворк:** **React Native**.
- **OCR:** **Google ML Kit — Text Recognition v2**, on-device, без интернета, бесплатно.
- iOS **вне скоупа** этой версии.

## 3. Экраны (поток)

| Экран | Назначение | Ключевое поведение |
|---|---|---|
| **Старт** | Логотип, кнопка «Начать осмотр», список незакрытых сессий | Тап по сессии → возврат в активную сессию |
| **Первое фото — номер** | Камера + рамка-подсказка пропорций номера | После снимка → OCR (≤2с) → «Верно» / «Переснять» |
| **Активная сессия** | Номер авто, счётчик файлов, сетка миниатюр (6–9), кнопки Фото/Видео, поле «Описание», красная кнопка «ЗАКОНЧИЛ» | «ЗАКОНЧИЛ» → диалог подтверждения → завершение |
| **Сессия завершена** | Галочка, номер, итог («8 фото, 1 видео»), «Новый осмотр» | Возврат на старт |

## 4. OCR — распознавание номера

### Форматы израильских номеров (оба должны распознаваться)
- Старый: `XXX-XX-XXX` (3 цифры — 2 цифры — 3 цифры).
- Новый: `XX-XXX-XX` (2 — 3 — 2).

> ⚠️ В ТЗ старый формат описан как «3 цифры — 2 буквы — 3 цифры», но пример (`123-45-678`) и стандарт израильских номеров — **только цифры**. При реализации исходить из числового формата; уточнить у заказчика, если потребуется поддержка букв.

### Логика
1. Снимок → ML Kit.
2. Из распознанного текста выбрать строку, подходящую под формат израильского номера.
3. Отформатировать в стандартный вид с дефисами.
4. Несколько кандидатов → выбрать наиболее уверенного.
5. Уверенность **< 85%** → запросить пересъёмку или ручной ввод.

### Обработка ошибок OCR
| Ситуация | Действие |
|---|---|
| Распознан уверенно | Показать номер, запросить подтверждение |
| Распознан неуверенно | Показать с предупреждением, запросить подтверждение |
| Не распознан | «Не удалось распознать», предложить переснять |
| Тёмно / размыто | Подсказка про освещение, кнопка «Переснять» |

## 5. Управление файлами

### Структура папок
```
/cases/
  /123-45-678/          ← название = номер авто
    plate.jpg           ← первое фото (номерной знак)
    photo_001.jpg
    photo_002.jpg
    video_001.mp4
    session.json        ← метаданные сессии
```

### session.json (схема)
```json
{
  "plate_number": "123-45-678",
  "session_start": "2026-05-25T09:14:00",
  "session_end": "2026-05-25T09:31:00",
  "mechanic_id": "user_042",
  "files": [
    { "name": "plate.jpg", "type": "photo", "timestamp": "09:14:22" },
    { "name": "photo_001.jpg", "type": "photo", "timestamp": "09:18:05" },
    { "name": "video_001.mp4", "type": "video", "timestamp": "09:22:11", "duration_sec": 34 }
  ],
  "description": "Трещина в блоке цилиндров. Замена блока.",
  "status": "closed"
}
```

### Загрузка
- Каждый файл загружается **немедленно** после съёмки.
- Есть WiFi → загрузка в реальном времени, статус «Загружено».
- Нет сети → файл кэшируется локально, статус «Ожидает загрузки».
- Сеть восстановилась → автоматическая загрузка очереди (в т.ч. при следующем запуске приложения).
- Индикатор (иконка облака): загружается / загружено / ошибка.

### Блокировка после «ЗАКОНЧИЛ»
- Статус сессии → `closed`, папка → **READ ONLY**.
- Добавление файлов невозможно ни из приложения, ни с сервера.
- Любая попытка изменить закрытую папку фиксируется в логе.

## 6. Технические требования к съёмке

**Фото:** ≥8 МП, JPEG, автофокус обязателен, вспышка авто.
**Видео:** ≥1080p, MP4 (H.264), макс. длина одного видео **3 минуты** → при превышении предупреждение + автостоп.

## 7. Уведомления

| Событие | Уведомление |
|---|---|
| Папка создана | «Кейс 123-45-678 открыт» |
| Файл загружен | Тихое обновление счётчика |
| Ошибка загрузки | «Нет подключения. Файл сохранён локально» |
| Сессия закрыта | «Кейс завершён. 8 файлов сохранено» |

После закрытия — автоуведомление на рабочую станцию **йоец шерута** (сервис-консультанта): «Механик завершил документирование кейса 123-45-678».

## 8. Безопасность

- Авторизация: логин + PIN или биометрия.
- Каждый файл хранит метаданные: кто, когда, с какого устройства.
- Файлы **не** попадают в галерею телефона — только внутри приложения.
- Локальный кэш **шифруется**.
- При смене сотрудника сессии предыдущего механика недоступны новому.

## 9. Вне скоупа (v1.0)

Интеграция с программой «Делек»; веб-интерфейс просмотра; аннотации фото (стрелки, пометки); AI-определение типа повреждения; iOS-версия.

## 10. Конвенции разработки

- Язык кода/комментариев: TypeScript (strict), функциональные компоненты + хуки.
- Навигация: `@react-navigation` (stack), 4 экрана из раздела 3.
- Структура: `src/screens`, `src/components`, `src/services` (camera, ocr, files, upload, auth), `src/store`, `src/types`.
- Камера: `react-native-vision-camera` (рекоменд.); OCR через нативный мост ML Kit.
- Файлы: `react-native-fs`; очередь загрузки и шифрование кэша — в `src/services`.
- Любая работа с файлами кейса соблюдает правило READ ONLY после `closed`.
- Тесты обязательны для OCR-парсинга номеров и логики жизненного цикла сессии.
