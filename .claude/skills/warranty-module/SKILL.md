# Warranty — гайдлайны модуля

Источник истины по деталям реализации. При конфликте с памятью — доверять этому файлу и `CLAUDE.md`.

## Лайфцикл сессии

Состояния: `open` (активна) -> `closed` (завершена, READ ONLY).

1. «Начать осмотр» -> экран первого фото.
2. Фото номера -> OCR -> подтверждение -> создаётся папка `/cases/<номер>/`, статус `open`, пишется `plate.jpg` + `session.json`.
3. Фото/видео в сессии -> файлы добавляются в папку, обновляется `files[]` и счётчик.
4. «ЗАКОНЧИЛ» -> диалог подтверждения -> `session_end` фиксируется, статус -> `closed`, папка -> READ ONLY.
5. Незакрытые (`open`) сессии показываются на стартовом экране и доступны для возврата.

**Правило READ ONLY:** в `closed` папку нельзя писать ни из приложения, ни с сервера. Любая попытка изменения логируется.

## Структура папки кейса

```
/cases/
  /123-45-678/          # имя папки = подтверждённый номер авто
    plate.jpg           # первое фото (номерной знак), всегда это имя
    photo_001.jpg       # фото, сквозная нумерация _001, _002, ...
    photo_002.jpg
    video_001.mp4       # видео, своя нумерация _001, _002, ...
    session.json        # метаданные (см. references/session.schema.json)
```

Имена: фото — `photo_NNN.jpg`, видео — `video_NNN.mp4`, нумерация с `001`, по три цифры, раздельные счётчики для фото и видео. Первое фото всегда `plate.jpg`.

## Форматы израильских номеров

| Тип | Шаблон | Пример |
|---|---|---|
| Старый | `XXX-XX-XXX` (3-2-3 цифры) | `123-45-678` |
| Новый | `XX-XXX-XX` (2-3-2 цифры) | `12-345-67` |

- На выходе номер всегда с дефисами в стандартном виде.
- Имя папки = отформатированный номер.
- Порог уверенности OCR: **85%**. Ниже -> пересъёмка или ручной ввод, без автосоздания папки.
- Примечание по ТЗ: старый формат там описан как «3 цифры — 2 буквы — 3 цифры», но пример и стандарт — цифры. По умолчанию числовой формат; поддержку букв уточнять у заказчика.

Готовый regex-помощник и парсер — `references/plate_formats.md`.

## session.json

Полная схема и пример — `references/session.schema.json`. Обязательные поля:
`plate_number`, `session_start`, `session_end` (null пока open), `mechanic_id`, `files[]`, `description`, `status`.
Каждый элемент `files[]`: `name`, `type` (`photo`|`video`), `timestamp`; для видео ещё `duration_sec`.

## Загрузка и офлайн

- Файл грузится немедленно после съёмки.
- Есть сеть -> realtime, статус «Загружено».
- Нет сети -> локальный (зашифрованный) кэш, статус «Ожидает загрузки», файл в очереди.
- Сеть вернулась / приложение перезапущено -> очередь догружается автоматически.
- Индикатор-облако на каждом файле: загружается / загружено / ошибка.

## Требования к съёмке

- Фото: >=8 МП, JPEG, автофокус обязателен, вспышка авто.
- Видео: >=1080p, MP4 (H.264), макс 3 минуты -> предупреждение + автостоп.

## Безопасность

- Авторизация: логин + PIN или биометрия.
- Метаданные в каждом файле: кто снял, когда, с какого устройства.
- Файлы только внутри приложения, НЕ в галерею.
- Локальный кэш шифруется.
- Сессии прежнего механика недоступны новому после смены сотрудника.

## Уведомления

Папка создана -> «Кейс <номер> открыт». Файл загружен -> тихо обновить счётчик. Ошибка загрузки -> «Нет подключения. Файл сохранён локально». Сессия закрыта -> «Кейс завершён. N файлов сохранено» + автоуведомление на станцию сервис-консультанта (йоец шерута).
