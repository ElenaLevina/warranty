# Передача файлов на ПК (локальная сеть)

После закрытия сессии (а также по мере съёмки) файлы кейса отправляются на
компьютер в локальной сети — вариант **A: HTTP-приёмник на ПК** (`server/`).

## Поток

1. После каждого снимка файл попадает в очередь (`StorageIndex`) и, если передача
   включена и есть сеть, сразу отправляется: на телефоне он **расшифровывается**
   (`CryptoService.openFile` → временный файл) и грузится на ПК; temp удаляется.
2. Нет сети / выключено → остаётся в очереди со статусом `pending`. При возврате
   сети (`NetworkUploadSync` через NetInfo) или на старте — `processQueue()` догружает.
3. По «ЗАКОНЧИЛ»: догрузка остатка + `POST …/complete` с `session.json` (это же
   закрывает §7 — уведомление рабочей станции). Сбой не блокирует закрытие — очередь
   повторит позже.
4. Индикаторы-облака в активной сессии показывают статус: ожидает / загрузка /
   загружено / ошибка.

Файлы на телефоне зашифрованы (Keystore), ключ не покидает устройство, поэтому
расшифровка делается **на телефоне перед отправкой** → на ПК лежат читаемые файлы.

## REST-контракт

Базовый адрес — `http://<IP-ПК>:<порт>` (вводится в **Настройки** приложения).
Все запросы: заголовок `Authorization: Bearer <token>`.

| Метод | Путь | Тело | Назначение |
|---|---|---|---|
| `GET`  | `/v1/health` | — | Проверка доступности (кнопка «Проверить соединение») |
| `POST` | `/v1/cases/{caseId}/files` | multipart: `file` + поля `filename`, `type` (`photo`/`video`), `sha256` | Один файл кейса |
| `POST` | `/v1/cases/{caseId}/complete` | `session.json` (JSON) | Кейс завершён |

- Идемпотентность: если файл с тем же именем и `sha256` уже есть — `200`, пропуск.
- Целостность: сервер сверяет `sha256`; несовпадение → `400`.
- Безопасность v1: bearer-токен по HTTP в доверенной LAN; TLS — задел на будущее.

## Настройка (коротко)

1. На ПК собрать и запустить `server/` (`-dir`, `-token`, `-addr`), открыть порт.
2. В приложении: **Старт → ⚙ Настройки** → включить передачу, ввести `http://<IP>:<порт>`
   и токен → «Проверить соединение» → «Сохранить».
