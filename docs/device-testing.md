# Тест на физическом телефоне — камера + OCR (Фаза 8)

Реальная камера и распознавание номера проверяются **только на телефоне**
(на эмуляторе нет нормального кадра номера). iOS вне скоупа.

## 1. Включить реальные возможности

В `src/app/featureFlags.ts`:
```ts
export const FEATURES = {
  realCamera: true,   // живая камера vision-camera (фото/видео)
  nativeOcr: true,    // распознавание номера через ML Kit
  nativeCrypto: false, // шифрование пока выкл (включим отдельно)
} as const;
```
> `realCamera` и `nativeOcr` независимы. Можно включить камеру без OCR (тогда сработает
> mock-OCR на реальном снимке) — удобно изолировать проблему, если что-то не так.

## 2. Собрать на устройстве

```bash
adb devices                     # телефон виден, статус 'device'
cd ~/dev/warranty/Warranty
npm run android                 # debug-сборка + установка
```
Первая сборка дольше: компилируются нативные модули (ML Kit, vision-camera, NDK/CMake).

> Изменения в `featureFlags.ts` — это JS, но переключение camera/native требует, чтобы
> нативные либы были в сборке. Если меняли только флаги после уже собранного APK —
> достаточно reload в Metro (`r`). Если впервые включили vision-camera/ML Kit — пересоберите.

## 3. Разрешения

При первом снимке Android запросит **Камеру** (и **Микрофон** для видео) — разрешите.
Манифест уже содержит `CAMERA`, `RECORD_AUDIO`, `camera.autofocus` (`AndroidManifest.xml`).
Если случайно отказали — Settings → Apps → Warranty → Permissions.

## 4. Чек-лист проверки

**Фото номера + OCR**
- [ ] Экран номера открывает живой preview с жёлтой рамкой.
- [ ] «Снять» → реальный кадр → индикатор «Распознаём…» → номер на экране.
- [ ] Корректный израильский номер форматируется в `XXX-XX-XXX` / `XX-XXX-XX`.
- [ ] Плохой кадр/нечитаемо → «Переснять» (не создаёт кейс).
- [ ] «✓ Верно» → создаётся папка кейса, переход в активную сессию.

**Фото в сессии**
- [ ] «📷 Фото» открывает камеру, снимок добавляется, счётчик растёт, файл `photo_NNN.jpg`.

**Видео**
- [ ] «🎥 Видео» → запись со счётчиком времени; повторное нажатие — стоп.
- [ ] Файл `video_NNN.mp4` добавлен, в `session.json` есть `duration_sec`.
- [ ] На 3:00 — авто-предупреждение и автостоп.

**Завершение / READ ONLY**
- [ ] «ЗАКОНЧИЛ» → подтверждение → экран итога «N фото, M видео».
- [ ] Кейс закрыт: `session.json.status = "closed"`, `session_end` проставлен.

## 5. Где смотреть файлы (sandbox)

```bash
adb shell run-as com.warranty ls -lR files/cases
adb exec-out run-as com.warranty cat files/cases/<номер>/session.json
```
(Файлы намеренно не попадают в галерею — ТЗ §8.)

## 6. Частые проблемы

| Симптом | Причина / решение |
|---|---|
| Сборка просит `react-native-worklets-core` | Проверить `VisionCamera_enableFrameProcessors=false` в `android/gradle.properties` |
| «Камера недоступна» на экране | Нет `useCameraDevice('back')` — проверить разрешение и что это реальный телефон |
| OCR всегда «Не удалось распознать» | ML Kit вернул пустой текст: плохой свет/фокус; проверить, что `nativeOcr=true` и модуль собран |
| Номер распознан, но `low_confidence` | Latin-распознаватель отдаёт слабую/NaN confidence (риск R3) — см. `native-modules.md`, при необходимости снизить порог/уточнить метрику |
| `run-as ... not debuggable` | Стоит release-сборка; пересобрать debug (`npm run android`) |

## 7. Что ещё не реальное (осознанно)

- **Шифрование at-rest** (`nativeCrypto=false`) — файлы пишутся в открытом виде в sandbox.
  Включение Keystore — отдельный шаг (`docs/native-modules.md`), после него для просмотра
  медиа понадобится `crypto.openFile` (decrypt во временный путь).
- **Разрешение фото/видео**: формат выбирается `useCameraFormat` под ~8 МП / 1080p, но на
  конкретном телефоне стоит свериться с фактическими размерами файла и при необходимости
  поджать список фильтров формата в `CameraCapture.tsx`.
- **Загрузка на сервер** — по-прежнему stub (v1.0 local-only).
